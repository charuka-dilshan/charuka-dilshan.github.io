Detailed Course Notes

1. What is IT Infrastructure?

[cite_start]

An **IT infrastructure** includes the hardware, software, and telecommunications equipment that, when combined, provide the underlying foundation to support the organization's goals[cite: 3250].

[cite_start]

It supports organizational strategies such as increasing productivity, enhancing decision-making, and enabling global reach [cite: 3268-3273].

2. Primary Infrastructure Components

A. Client/Server Network

[cite_start]

A network in which one or more computers are **servers** (providing services) and the other computers are **clients** (requesting services) [cite: 3283-3284].

B. N-Tier Infrastructures

    [cite_start]
  • 2-Tier Infrastructure: The most basic setup, containing only two tiers: the client and the server [cite: 3306-3307].
  • 3-Tier Infrastructure: Contains clients, **application servers**, and **data servers**. [cite_start]The application server handles the business logic, separating it from the data [cite: 3308-3309].

3. Additional IT Elements: Information Views

[cite_start]

This category is responsible for the presentation of information to users[cite: 3338].

    [cite_start]
  • Intranet: An internal organizational Internet that is guarded against outside access by a **firewall**[cite: 3347].
    • [cite_start]
  • Extranet: An intranet that is restricted to an organization *and* certain outsiders, such as customers and suppliers[cite: 3348].
    • [cite_start]
  • Enterprise Information Portal (EIP): Allows knowledge workers to access company information via a Web interface[cite: 3350].
    • [cite_start]
  • Digital Dashboard: Displays key information gathered from several sources on a single screen, tailored to an individual's needs [cite: 3354-3355].

4. Additional IT Elements: Business Logic

[cite_start]

This category is responsible for maintaining business rules and protecting information [cite: 3426-3428].

    [cite_start]
  • Integration: Allows separate applications to communicate directly with each other (e.g., an Order Entry system automatically sending info to the Billing system)[cite: 3436].
  • Web Services: Technologies used to transmit and process information on a network. [cite_start]Key to **interoperability** (different systems talking to each other) [cite: 3455-3456].
    • Microsoft .NET: Microsoft's version of Web services.
    • J2EE (Java 2 Enterprise Edition): Sun Microsystems' version.
    • [cite_start]
  • Workflow Systems: Defines and automates all the steps or business rules required for a process to run correctly [cite: 3472-3473].
    • [cite_start]
  • Application Service Provider (ASP): Supplies software applications over the Internet that would otherwise reside on a customer's in-house computer[cite: 3476].

5. Additional IT Elements: Data Storage

[cite_start]

This category is responsible for data storage and manipulation [cite: 3480-3481].

[cite_start] [cite_start] [cite_start]

6. Supporting the Infrastructure

A. Backup and Recovery

    [cite_start]
  • Backup: The process of making a copy of the information stored on a computer[cite: 3531].
    • [cite_start]
  • Recovery: The process of reinstalling the backup information in the event the information was lost[cite: 3532].

B. Disaster Recovery Plan

[cite_start]

A detailed process for recovering an IT system in the event of a catastrophic disaster (e.g., fire, flood)[cite: 3535].

    [cite_start]
  • Hot Site: A separate and **fully equipped** facility where the company can move immediately and resume business[cite: 3538].
    • [cite_start]
  • Cold Site: A separate facility that does **not have any computer equipment**, but is a place where workers can move[cite: 3539].

C. The "-ilities" (System Qualities)

[cite_start]

These are factors to consider when developing an IT infrastructure [cite: 3541-3542]:

  • Availability: When the system will be available for access.
  • Accessibility: Who has the right to access different types of information.
  • Reliability: Ensures the system is functioning correctly and providing accurate information.
  • Scalability: How well the system can adapt to increased demands.
  • Flexibility: The system's ability to change quickly.
  • Performance: How quickly the system performs a process.
  • Capacity Planning: Determines future infrastructure requirements.
